A Hands-On RDKit Tutorial آموزش عملی RDKit

RDKit is the leading open-source toolkit for cheminformatics. This guide walks you all the way from installing the library to reading files, computing descriptors, searching substructures, measuring similarity, generating 3D structures, running reactions, and filtering drug-like molecules — with runnable Python at every step. RDKit پرکاربردترین کتابخانه‌ی متن‌باز برای شیمی‌انفورماتیک است. این راهنما شما را از نصب کتابخانه تا خواندن فایل‌ها، محاسبه‌ی توصیفگرها، جستجوی زیرساختارها، اندازه‌گیری شباهت، تولید ساختار سه‌بعدی، اجرای واکنش‌ها و فیلترکردن مولکول‌های شبه‌دارو همراهی می‌کند — با کد پایتونِ قابل‌اجرا در هر گام.

Python RDKit Cheminformatics Drug Discovery Machine Learning

1Introductionمقدمه

Think of RDKit as a bridge between chemistry and code. A molecule that a chemist draws on paper becomes, inside RDKit, a graph of atoms and bonds you can query, transform, and turn into numbers. Once a structure lives inside a Mol object, you can ask it almost anything: how heavy is it, does it contain a benzene ring, how similar is it to a known drug, what does it look like in 3D. This is exactly the workflow that powers modern drug discovery, virtual screening, and machine-learning models for chemistry.

RDKit را پُلی میان شیمی و کد در نظر بگیرید. مولکولی که یک شیمی‌دان روی کاغذ می‌کشد، درون RDKit به گرافی از اتم‌ها و پیوندها تبدیل می‌شود که می‌توانید آن را پرس‌وجو، دستکاری و به عدد تبدیل کنید. وقتی یک ساختار در قالب یک شیء Mol قرار گرفت، می‌توانید تقریباً هر چیزی از آن بپرسید: چقدر سنگین است، آیا حلقه‌ی بنزنی دارد، چقدر به یک داروی شناخته‌شده شبیه است، در سه‌بعد چه شکلی است. این دقیقاً همان گردش‌کاری است که کشف داروی مدرن، غربالگری مجازی و مدل‌های یادگیری ماشین برای شیمی را پیش می‌برد.

A quick note on notation: throughout the tutorial we use SMILES (a text string that spells out a molecule, e.g. CCO for ethanol) and SMARTS (a pattern language for querying molecules, like regular expressions for chemistry). 💡 پیش‌نیاز: پایتونِ مقدماتی. پیش‌زمینه‌ی رسمی شیمی لازم نیست — مفاهیم شیمیایی را در جای خود توضیح می‌دهیم. نکته‌ای درباره‌ی نمادگذاری: در سراسر آموزش از SMILES (رشته‌ای متنی که یک مولکول را هجی می‌کند، مثلاً CCO برای اتانول) و SMARTS (زبانی الگویی برای پرس‌وجوی مولکول‌ها، مانند عبارات باقاعده برای شیمی) استفاده می‌کنیم.

2Installationنصب

RDKit dropped Python 2 support back in the 2019.03 release, so make sure you're on Python 3. Both pip and conda work today; conda has historically been the most reliable because it pulls in all the compiled dependencies for you.

RDKit از نسخه‌ی ۲۰۱۹٫۰۳ پشتیبانی از پایتون ۲ را کنار گذاشته است، پس مطمئن شوید روی پایتون ۳ هستید. امروزه هم pip و هم conda کار می‌کنند؛ conda از نظر تاریخی قابل‌اعتمادترین بوده چون همه‌ی وابستگی‌های کامپایل‌شده را برایتان می‌آورد.

bash 
# Using pip (works well on modern systems)
pip install rdkit

# Or using conda (recommended — pulls all compiled deps)
conda install -c conda-forge rdkit

# Verify the install
python -c "from rdkit import Chem; print(Chem.MolFromSmiles('CCO').GetNumAtoms())"
# -> 3
Almost everything you need lives in rdkit.Chem. Heavier or less-common machinery (3D embedding, force fields, reactions) lives in rdkit.Chem.AllChem. This split keeps import times fast — you only load the big stuff when you actually need it. تقریباً هرچیزی که لازم دارید در rdkit.Chem قرار دارد. ابزارهای سنگین‌تر یا کم‌کاربردتر (تعبیه‌ی سه‌بعدی، میدان‌های نیرو، واکنش‌ها) در rdkit.Chem.AllChem هستند. این تفکیک زمان وارد کردن را سریع نگه می‌دارد — بخش‌های بزرگ را فقط وقتی واقعاً نیاز دارید بارگذاری می‌کنید.

3Reading & Building Moleculesخواندن و ساخت مولکول

Everything starts with a Mol object. You can create one from a SMILES string, from a single-molecule Mol file, or from a raw Mol block already sitting in a Python string. Each function returns a molecule on success, or None on failure — RDKit never raises an exception for a bad structure, so you always check the return value yourself.

همه‌چیز با یک شیء Mol آغاز می‌شود. می‌توانید آن را از یک رشته‌ی SMILES، از یک فایل Mol تک‌مولکولی، یا از یک بلاک Mol خام که در یک رشته‌ی پایتون قرار دارد بسازید. هر تابع در صورت موفقیت یک مولکول و در صورت شکست None برمی‌گرداند — RDKit هرگز برای یک ساختار خراب استثنا پرتاب نمی‌کند، پس همیشه خودتان مقدار بازگشتی را بررسی می‌کنید.

python 
from rdkit import Chem

# Three ways to build one molecule
m = Chem.MolFromSmiles('Cc1ccccc1')                 # from SMILES (toluene)
m = Chem.MolFromMolFile('data/input.mol')           # from a .mol file
block = open('data/input.mol').read()
m = Chem.MolFromMolBlock(block)                     # from a Mol block string

# Success gives a Mol; failure gives None (never an exception)
bad = Chem.MolFromMolFile('data/invalid.mol')
print(bad is None)                                  # True

Sensible error messages

پیام‌های خطای معنادار

When a SMILES can't possibly describe a real molecule, RDKit prints a helpful diagnostic and returns None. For example, an oxygen with too many bonds, or an aromatic ring that can't be "kekulized" (assigned alternating single/double bonds):

وقتی یک SMILES نمی‌تواند مولکول واقعی را توصیف کند، RDKit یک پیام تشخیصی مفید چاپ کرده و None برمی‌گرداند. برای مثال، اکسیژنی با پیوندهای بیش‌ازحد، یا حلقه‌ی آروماتیکی که نمی‌توان آن را «کِکوله» کرد (تخصیص پیوندهای یگانه/دوگانه‌ی متناوب):

python 
m1 = Chem.MolFromSmiles('CO(C)C')   # O with too many bonds
# [12:18:01] Explicit valence for atom # 1 O greater than permitted
print(m1 is None)                   # True

m2 = Chem.MolFromSmiles('c1cc1')    # can't kekulize a 3-membered aromatic ring
# [12:20:41] Can't kekulize mol
print(m2 is None)                   # True
⚠️ Because errors come back as None, forgetting the null check is the single most common RDKit bug. Get in the habit of writing if mol is None: continue whenever you loop over parsed structures. ⚠️ چون خطاها به‌صورت None بازمی‌گردند، فراموش کردن بررسیِ نال رایج‌ترین باگ RDKit است. عادت کنید هرجا روی ساختارهای تجزیه‌شده پیمایش می‌کنید، if mol is None: continue بنویسید.

4Working with Molecule Setsکار با مجموعه‌ای از مولکول‌ها

Real projects rarely deal with a single molecule — you read whole libraries of them. RDKit calls its file readers Suppliers. An SDMolSupplier reads an SDF file, and you can loop over it, turn it into a list, or index into it like an array.

پروژه‌های واقعی به‌ندرت با یک مولکول سروکار دارند — شما کتابخانه‌های کاملی از آن‌ها را می‌خوانید. RDKit خواننده‌های فایلش را Supplier می‌نامد. یک SDMolSupplier فایل SDF را می‌خواند و می‌توانید روی آن پیمایش کنید، به لیست تبدیلش کنید یا مثل یک آرایه به آن اندیس بزنید.

python 
from rdkit import Chem

# Best practice: use a context manager AND check each molecule
with Chem.SDMolSupplier('data/5ht3ligs.sdf') as suppl:
    for mol in suppl:
        if mol is None:        # a corrupt record parses to None
            continue
        print(mol.GetNumAtoms())

# A supplier is also a random-access object
suppl = Chem.SDMolSupplier('data/5ht3ligs.sdf')
print(len(suppl))              # how many records
print(suppl[0].GetNumAtoms())  # jump straight to the first one
mols = [m for m in suppl if m is not None]  # materialize to a list

Streaming and compressed files

فایل‌های جریانی و فشرده

If your file is huge or gzip-compressed, use a ForwardSDMolSupplier. It reads from any file-like object one record at a time, which means it can stream straight out of a .sdf.gz without unpacking the whole thing. The trade-off: it's forward-only, so no indexing. For very large files, the multithreaded suppliers can parse records in parallel.

اگر فایلتان بزرگ یا با gzip فشرده باشد، از ForwardSDMolSupplier استفاده کنید. این از هر شیء فایل‌مانند، هر بار یک رکورد را می‌خواند، یعنی می‌تواند مستقیماً از یک .sdf.gz بدون باز کردن کل آن جریان بگیرد. بده‌بستان: فقط رو‌به‌جلو است، پس اندیس‌گذاری ندارد. برای فایل‌های بسیار بزرگ، Supplierهای چندنخی می‌توانند رکوردها را به‌موازات تجزیه کنند.

python 
import gzip
from rdkit import Chem

# Stream molecules directly out of a gzip-compressed SDF
inf = gzip.open('data/actives_5ht3.sdf.gz')
with Chem.ForwardSDMolSupplier(inf) as fsuppl:
    mols = [m for m in fsuppl if m is not None]
print(len(mols))               # e.g. 180

# For very large files, parse in parallel with a multithreaded supplier
n = 0
with Chem.MultithreadedSDMolSupplier('data/5ht3ligs.sdf') as ms:
    for mol in ms:
        if mol is not None:
            n += 1
# Note: output order is NOT guaranteed with multithreading

5Writing & Saving Moleculesنوشتن و ذخیره‌ی مولکول‌ها

Going the other way — molecule to text — is just as easy. The workhorse is MolToSmiles, which produces a canonical SMILES: no matter how you originally wrote the molecule, the same structure always yields the same string. That's what makes SMILES usable as a dictionary key or a duplicate-detector.

راه معکوس — مولکول به متن — به همان اندازه ساده است. اسب‌بارکشِ اصلی MolToSmiles است که یک SMILES متعارف (canonical) تولید می‌کند: مهم نیست مولکول را در ابتدا چطور نوشته‌اید، یک ساختار یکسان همیشه رشته‌ی یکسانی می‌دهد. همین است که SMILES را به‌عنوان کلید دیکشنری یا آشکارساز تکراری قابل‌استفاده می‌کند.

python 
from rdkit import Chem

# Canonical SMILES: three different inputs, one output
for s in ('C1=CC=CN=C1', 'c1cccnc1', 'n1ccccc1'):
    print(Chem.MolToSmiles(Chem.MolFromSmiles(s)))   # all -> 'c1ccncc1'

# Drop stereochemistry if you don't need it
m = Chem.MolFromMolFile('data/chiral.mol')
print(Chem.MolToSmiles(m))                       # 'C[C@H](O)c1ccccc1'
print(Chem.MolToSmiles(m, isomericSmiles=False)) # 'CC(O)c1ccccc1'

Writing many molecules to an SDF

نوشتن چند مولکول در یک SDF

To save a whole set, use an SDWriter. You can name each molecule by setting its special _Name property, and RDKit will auto-generate 2D coordinates on the fly so the Mol block is drawable.

برای ذخیره‌ی یک مجموعه‌ی کامل از SDWriter استفاده کنید. می‌توانید هر مولکول را با تنظیم ویژگی خاص _Name نام‌گذاری کنید، و RDKit به‌صورت خودکار مختصات دوبعدی تولید می‌کند تا بلاک Mol قابل‌رسم باشد.

python 
from rdkit import Chem
from rdkit.Chem import AllChem

m = Chem.MolFromSmiles('C1CCC1')      # cyclobutane
m.SetProp('_Name', 'cyclobutane')     # names show up in the Mol block
AllChem.Compute2DCoords(m)            # give it a 2D depiction

with Chem.SDWriter('out.sdf') as w:   # context manager closes the file for you
    w.write(m)

# Other writers exist too: SmilesWriter, TDTWriter

Pickle: the fast lane

Pickle: مسیر سریع

🚀 Performance tip: Re-parsing SDF or SMILES is slow. If you'll reuse the same molecules repeatedly, pickle them once and load the pickle instead. In the RDKit docs' own benchmark, loading 699 molecules took 10.8 s from SDF but only 0.7 s from a pickle — and the pickle file was a third the size. Use pickle.dumps(mol) / pickle.loads(...), or the leaner mol.ToBinary() / Chem.Mol(binStr). 🚀 نکته‌ی کارایی: تجزیه‌ی دوباره‌ی SDF یا SMILES کند است. اگر قرار است همان مولکول‌ها را مکرراً استفاده کنید، یک‌بار آن‌ها را pickle کنید و به‌جای تجزیه، pickle را بارگذاری کنید. در محک خودِ مستندات RDKit، بارگذاری ۶۹۹ مولکول ۱۰٫۸ ثانیه از SDF اما فقط ۰٫۷ ثانیه از pickle طول کشید — و فایل pickle یک‌سومِ اندازه بود. از pickle.dumps(mol) / pickle.loads(...) یا از mol.ToBinary() / Chem.Mol(binStr)ِ سبک‌تر استفاده کنید.

6Atoms, Bonds & Ringsاتم‌ها، پیوندها و حلقه‌ها

A molecule is a graph, and RDKit lets you walk it freely. You can loop over every atom and bond, grab one by index, ask an atom for its neighbors, or query the ring system. This graph-level access is where a lot of custom cheminformatics logic lives.

یک مولکول یک گراف است، و RDKit اجازه می‌دهد آزادانه روی آن حرکت کنید. می‌توانید روی هر اتم و پیوند پیمایش کنید، یکی را با اندیس بگیرید، از یک اتم همسایه‌هایش را بپرسید، یا سیستم حلقه را پرس‌وجو کنید. این دسترسی در سطح گراف جایی است که بخش زیادی از منطق سفارشی شیمی‌انفورماتیک زندگی می‌کند.

python 
from rdkit import Chem

m = Chem.MolFromSmiles('C1OC1')   # ethylene oxide

# Walk atoms and bonds
for atom in m.GetAtoms():
    print(atom.GetIdx(), atom.GetSymbol(), atom.GetAtomicNum())
print(m.GetBonds()[0].GetBondType())          # SINGLE

# Access individual pieces
print(m.GetAtomWithIdx(0).GetSymbol())        # 'C'
print(m.GetBondBetweenAtoms(0, 1).GetBondType())

# Atoms know their neighbors
a = m.GetAtomWithIdx(0)
print([nbr.GetSymbol() for nbr in a.GetNeighbors()])  # ['O', 'C']

Ring information

اطلاعات حلقه

Atoms and bonds carry ring membership info. For repeated queries, ask the molecule for its RingInfo object rather than calling GetAtomWithIdx in a loop — it's much faster.

اتم‌ها و پیوندها اطلاعات عضویت در حلقه را حمل می‌کنند. برای پرس‌وجوهای مکرر، به‌جای فراخوانی GetAtomWithIdx در حلقه، از مولکول شیء RingInfo آن را بخواهید — بسیار سریع‌تر است.

python 
m = Chem.MolFromSmiles('OC1C2C1CC2')

print(m.GetAtomWithIdx(0).IsInRing())        # False (the hydroxyl O)
print(m.GetAtomWithIdx(2).IsInRingSize(3))   # True
print(m.GetAtomWithIdx(2).IsInRingSize(4))   # True

# Efficient bulk ring queries
ri = m.GetRingInfo()
print(ri.NumAtomRings(2))                    # 2  (atom 2 is in two rings)
print(ri.IsAtomInRingOfSize(1, 3))           # True

# The smallest set of smallest rings (SSSR)
ssr = Chem.GetSymmSSSR(m)
print(len(ssr), [list(r) for r in ssr])

7Modifying Moleculesاصلاح مولکول‌ها

By default RDKit keeps hydrogens implicit — they're counted but not drawn as separate nodes in the graph, which keeps things tidy. When you need explicit Hs (essential for realistic 3D geometry), add them with AddHs and strip them later with RemoveHs.

به‌طور پیش‌فرض RDKit هیدروژن‌ها را ضمنی نگه می‌دارد — شمرده می‌شوند اما به‌عنوان گره‌های جداگانه در گراف رسم نمی‌شوند، که کارها را مرتب نگه می‌دارد. وقتی به هیدروژن‌های صریح نیاز دارید (برای هندسه‌ی واقع‌گرایانه‌ی سه‌بعدی ضروری است)، آن‌ها را با AddHs اضافه کرده و بعداً با RemoveHs حذف کنید.

python 
from rdkit import Chem

m = Chem.MolFromSmiles('CCO')
print(m.GetNumAtoms())          # 3 (Hs are implicit)

m2 = Chem.AddHs(m)
print(m2.GetNumAtoms())         # 9 (Hs now explicit)

m3 = Chem.RemoveHs(m2)
print(m3.GetNumAtoms())         # 3 again

Aromaticity, Kekulization & Sanitization

آروماتیسیته، کِکوله‌سازی و پاک‌سازی

RDKit stores aromatic rings with a special "aromatic" bond type. Kekulize converts them into alternating single/double bonds; SanitizeMol restores the aromatic perception and, more importantly, validates that the whole molecule is chemically consistent. Sanitization is your safety net after any structural edit.

RDKit حلقه‌های آروماتیک را با یک نوع پیوند «آروماتیک» ویژه ذخیره می‌کند. Kekulize آن‌ها را به پیوندهای متناوب یگانه/دوگانه تبدیل می‌کند؛ SanitizeMol درک آروماتیک را بازیابی و مهم‌تر از آن، سازگاری شیمیایی کل مولکول را اعتبارسنجی می‌کند. پاک‌سازی تور ایمنی شما پس از هر ویرایش ساختاری است.

python 
from rdkit import Chem

m = Chem.MolFromSmiles('c1ccccc1')            # benzene
print(m.GetBondWithIdx(0).GetBondType())      # AROMATIC

Chem.Kekulize(m, clearAromaticFlags=True)
print(m.GetBondWithIdx(0).GetBondType())      # SINGLE
print(m.GetBondWithIdx(1).GetBondType())      # DOUBLE

Chem.SanitizeMol(m)                           # restore aromatic perception
print(m.GetBondWithIdx(0).GetBondType())      # AROMATIC again
⚠️ Editing atoms directly (e.g. changing an element) can create chemical nonsense that looks fine until you sanitize. After any hand edit, always call Chem.SanitizeMol(mol) — if the structure is impossible, it will raise an exception rather than silently producing garbage. ⚠️ ویرایش مستقیم اتم‌ها (مثلاً تغییر یک عنصر) می‌تواند مهملِ شیمیایی بسازد که تا وقتی پاک‌سازی نکنید سالم به‌نظر می‌رسد. پس از هر ویرایش دستی، همیشه Chem.SanitizeMol(mol) را فراخوانی کنید — اگر ساختار ناممکن باشد، به‌جای تولید بی‌سروصدای زباله، استثنا پرتاب می‌کند.

8Molecular Descriptorsتوصیفگرهای مولکولی

Descriptors turn a molecule into numbers — molecular weight, LogP (lipophilicity), hydrogen-bond donors/acceptors, topological polar surface area (TPSA), and 200+ more. They're the raw features for QSAR and machine-learning models.

توصیفگرها یک مولکول را به عدد تبدیل می‌کنند — وزن مولکولی، LogP (چربی‌دوستی)، دهنده/گیرنده‌های پیوند هیدروژنی، مساحت سطح قطبی توپولوژیک (TPSA) و بیش از ۲۰۰ مورد دیگر. این‌ها ویژگی‌های خام برای مدل‌های QSAR و یادگیری ماشین هستند.

python 
from rdkit import Chem
from rdkit.Chem import Descriptors

mol = Chem.MolFromSmiles('CC(=O)OC1=CC=CC=C1C(=O)O')  # aspirin

print('MolWt:', Descriptors.MolWt(mol))
print('LogP:', Descriptors.MolLogP(mol))
print('H-bond donors:', Descriptors.NumHDonors(mol))
print('H-bond acceptors:', Descriptors.NumHAcceptors(mol))
print('TPSA:', Descriptors.TPSA(mol))

All descriptors at once → pandas

همه‌ی توصیفگرها یک‌جا ← pandas

Instead of calling descriptors one by one, CalcMolDescriptors returns every available descriptor as a dictionary — perfect for building a feature matrix. Feed a list of these dicts straight into a pandas DataFrame and you have a ready-to-model table.

به‌جای فراخوانی تک‌تک توصیفگرها، CalcMolDescriptors هر توصیفگر موجود را به‌صورت یک دیکشنری برمی‌گرداند — عالی برای ساخت ماتریس ویژگی. یک لیست از این دیکشنری‌ها را مستقیم به یک DataFrame پانداس بدهید و یک جدولِ آماده‌ی مدل‌سازی دارید.

python 
import pandas as pd
from rdkit import Chem
from rdkit.Chem import Descriptors

smiles = ['CCO', 'CC(=O)O', 'c1ccccc1', 'CC(=O)OC1=CC=CC=C1C(=O)O']
mols = [Chem.MolFromSmiles(s) for s in smiles]

# One dict of ~200 descriptors per molecule
rows = [Descriptors.CalcMolDescriptors(m) for m in mols]
df = pd.DataFrame(rows)
print(df.shape)          # (4, 208) — instant feature matrix
print(df[['MolWt', 'MolLogP', 'TPSA']])

Partial charges

بارهای جزئی

Per-atom Gasteiger partial charges are computed in place and stored as an atom property you read back afterward.

بارهای جزئی گاستایگر برای هر اتم در جای خود محاسبه و به‌عنوان یک ویژگی اتمی ذخیره می‌شوند که بعداً بازخوانی می‌کنید.

python 
from rdkit.Chem import AllChem

m = Chem.MolFromSmiles('c1ccccc1C(=O)O')
AllChem.ComputeGasteigerCharges(m)
q = m.GetAtomWithIdx(0).GetDoubleProp('_GasteigerCharge')
print(round(q, 3))       # e.g. -0.047

9Substructure Searchجستجوی زیرساختار

Substructure search answers "does this molecule contain that pattern?" — the backbone of library filtering. Patterns are usually written in SMARTS, a richer cousin of SMILES designed for querying. HasSubstructMatch gives a yes/no; GetSubstructMatches gives the actual matching atom indices.

جستجوی زیرساختار به این پرسش پاسخ می‌دهد که «آیا این مولکول شامل آن الگو هست؟» — ستون فقرات فیلترکردن کتابخانه. الگوها معمولاً با SMARTS نوشته می‌شوند، پسرعموی غنی‌ترِ SMILES که برای پرس‌وجو طراحی شده است. HasSubstructMatch پاسخ بله/خیر می‌دهد؛ GetSubstructMatches اندیس‌های اتمیِ منطبق را می‌دهد.

python 
from rdkit import Chem

m = Chem.MolFromSmiles('c1ccccc1O')     # phenol
patt = Chem.MolFromSmarts('ccO')        # aromatic-C, aromatic-C, O

print(m.HasSubstructMatch(patt))        # True
print(m.GetSubstructMatch(patt))        # (0, 5, 6) — one match
print(m.GetSubstructMatches(patt))      # ((0, 5, 6), (4, 5, 6)) — all matches

# Filter a whole library in one comprehension
patt = Chem.MolFromSmarts('c[NH1]')
with Chem.SDMolSupplier('data/actives_5ht3.sdf') as suppl:
    hits = [m for m in suppl if m is not None and m.HasSubstructMatch(patt)]
print(len(hits))

SMARTS vs SMILES semantics

معنای SMARTS در برابر SMILES

You can query with either a SMARTS or a SMILES pattern, but they don't mean the same thing. SMILES is literal about aromaticity; SMARTS lets you be precise. Notice how a lowercase c (aromatic carbon) matters:

می‌توانید با الگوی SMARTS یا SMILES پرس‌وجو کنید، اما معنای یکسانی ندارند. SMILES درباره‌ی آروماتیسیته تحت‌اللفظی است؛ SMARTS اجازه‌ی دقت می‌دهد. توجه کنید که چطور یک cی کوچک (کربن آروماتیک) اهمیت دارد:

python 
m = Chem.MolFromSmiles('C1=CC=CC=C1OC')   # anisole (aromatic ring + OMe)

print(m.HasSubstructMatch(Chem.MolFromSmiles('COC')))   # True
print(m.HasSubstructMatch(Chem.MolFromSmarts('COC')))   # False (needs aromatic c)
print(m.HasSubstructMatch(Chem.MolFromSmarts('COc')))   # True

# Stereochemistry is ignored by default — opt in with useChirality
m = Chem.MolFromSmiles('CC[C@H](F)Cl')
print(m.HasSubstructMatch(Chem.MolFromSmiles('C[C@@H](F)Cl')))                    # True
print(m.HasSubstructMatch(Chem.MolFromSmiles('C[C@@H](F)Cl'), useChirality=True)) # False

10Chemical Transformationsتبدیل‌های شیمیایی

Once you can match a substructure, you can edit around it. RDKit ships quick helpers to delete a group, replace it, or split a molecule into a scaffold and its side chains — the everyday moves of SAR analysis.

وقتی توانستید یک زیرساختار را منطبق کنید، می‌توانید پیرامون آن ویرایش کنید. RDKit کمک‌کننده‌های سریعی برای حذف یک گروه، جایگزینی آن، یا تقسیم یک مولکول به داربست و شاخه‌های جانبی‌اش دارد — حرکت‌های روزمره‌ی تحلیل SAR.

python 
from rdkit import Chem
from rdkit.Chem import AllChem

# Delete a substructure
m = Chem.MolFromSmiles('CC(=O)O')
rm = AllChem.DeleteSubstructs(m, Chem.MolFromSmarts('C(=O)[OH]'))
print(Chem.MolToSmiles(rm))          # 'C'

# Replace a substructure
m = Chem.MolFromSmiles('CC(=O)N')
rep = AllChem.ReplaceSubstructs(m, Chem.MolFromSmarts('[$(NC(=O))]'),
                                Chem.MolFromSmiles('OC'))
print(Chem.MolToSmiles(rep[0]))      # 'COC(C)=O'

# Split into core + side chains (SAR tables)
m1 = Chem.MolFromSmiles('BrCCc1cncnc1C(=O)O')
core = Chem.MolFromSmiles('c1cncnc1')
print(Chem.MolToSmiles(Chem.ReplaceCore(m1, core)))  # '[1*]CCBr.[2*]C(=O)O'

Murcko scaffolds

داربست‌های مورکو

A Murcko scaffold is the ring system of a molecule with its side chains removed — the "core shape." It's the standard way to group compounds by their skeleton. You can even reduce it further to a generic framework where every atom becomes carbon and every bond single.

داربست مورکو، سیستم حلقه‌ی یک مولکول با حذف شاخه‌های جانبی است — «شکلِ هسته». این روش استاندارد گروه‌بندی ترکیبات بر اساس اسکلتشان است. حتی می‌توانید آن را به یک چارچوب عام تقلیل دهید که در آن هر اتم کربن و هر پیوند یگانه می‌شود.

python 
from rdkit import Chem
from rdkit.Chem.Scaffolds import MurckoScaffold

m = Chem.MolFromSmiles('CC(=O)Nc1ccc(O)cc1')      # paracetamol
core = MurckoScaffold.GetScaffoldForMol(m)
print(Chem.MolToSmiles(core))                     # the ring core
generic = MurckoScaffold.MakeScaffoldGeneric(core)
print(Chem.MolToSmiles(generic))                  # all-carbon skeleton

11Fingerprints & Similarityاثرانگشت و شباهت

A fingerprint encodes a molecule's features into a bit vector (or count vector), so two molecules can be compared numerically. The modern, recommended way is to create a fingerprint generator once and reuse it. The Tanimoto coefficient (0 = totally different, 1 = identical fingerprints) is the standard similarity metric.

یک اثرانگشت، ویژگی‌های یک مولکول را در یک بردار بیتی (یا بردار شمارشی) رمزگذاری می‌کند تا دو مولکول به‌صورت عددی مقایسه شوند. راه مدرن و توصیه‌شده این است که یک‌بار یک مولد اثرانگشت بسازید و دوباره استفاده کنید. ضریب تنیموتو (۰ = کاملاً متفاوت، ۱ = اثرانگشت‌های یکسان) معیار استاندارد شباهت است.

python 
from rdkit import Chem, DataStructs
from rdkit.Chem import AllChem

ms = [Chem.MolFromSmiles(s) for s in ('CCOC', 'CCO', 'COC')]

# Modern approach: build a generator, reuse it
fpgen = AllChem.GetRDKitFPGenerator()
fps = [fpgen.GetFingerprint(m) for m in ms]

print(DataStructs.TanimotoSimilarity(fps[0], fps[1]))  # ~0.6
print(DataStructs.DiceSimilarity(fps[0], fps[1]))      # ~0.75
# Other metrics: Cosine, Sokal, Russel, Kulczynski, Tversky ...

Morgan (circular) fingerprints — the ECFP workhorse

اثرانگشت مورگان (دایره‌ای) — اسب‌بارکشِ ECFP

Morgan fingerprints (a.k.a. ECFP/FCFP) are the most popular choice for similarity and ML. They describe each atom's circular neighborhood out to a given radius. Remember: RDKit's radius=2 corresponds to ECFP4 (the 4 is the diameter). You can get them as bit vectors or as counts.

اثرانگشت‌های مورگان (معروف به ECFP/FCFP) محبوب‌ترین انتخاب برای شباهت و یادگیری ماشین‌اند. آن‌ها همسایگی دایره‌ای هر اتم را تا یک radius معین توصیف می‌کنند. به‌یاد داشته باشید: radius=2ی RDKit معادل ECFP4 است (عدد ۴ قطر است). می‌توانید آن‌ها را به‌صورت بردار بیتی یا شمارشی بگیرید.

python 
from rdkit import Chem, DataStructs
from rdkit.Chem import AllChem

m1 = Chem.MolFromSmiles('Cc1ccccc1')   # toluene
m2 = Chem.MolFromSmiles('Cc1ncccc1')   # methylpyridine

fpgen = AllChem.GetMorganGenerator(radius=2)   # ~ ECFP4
fp1 = fpgen.GetFingerprint(m1)                 # bit vector (2048 bits)
fp2 = fpgen.GetFingerprint(m2)
print(DataStructs.TanimotoSimilarity(fp1, fp2))

# Counts instead of bits
c1 = fpgen.GetSparseCountFingerprint(m1)

# Feature-based (FCFP-style) invariants pharmacophore-like similarity
inv = AllChem.GetMorganFeatureAtomInvGen()
ffpgen = AllChem.GetMorganGenerator(radius=2, atomInvariantsGenerator=inv)

MACCS keys & explaining bits

کلیدهای MACCS و توضیح بیت‌ها

The 166 public MACCS keys are a fixed set of interpretable structural questions ("does it have a carboxylic acid?"). And with the generators' additionalOutput, you can even trace which atoms set a given bit — invaluable for explaining a model.

۱۶۶ کلید عمومی MACCS مجموعه‌ای ثابت از پرسش‌های ساختاری قابل‌تفسیرند («آیا کربوکسیلیک‌اسید دارد؟»). و با additionalOutputی مولدها، حتی می‌توانید ردیابی کنید کدام اتم‌ها یک بیت مشخص را روشن کرده‌اند — بی‌نظیر برای توضیح یک مدل.

python 
from rdkit.Chem import MACCSkeys

ms = [Chem.MolFromSmiles(s) for s in ('CCOC', 'CCO', 'COC')]
fps = [MACCSkeys.GenMACCSKeys(m) for m in ms]
print(DataStructs.TanimotoSimilarity(fps[0], fps[1]))   # 0.5

# Which atoms set which Morgan bit?
m = Chem.MolFromSmiles('c1cccnc1C')
fpgen = AllChem.GetMorganGenerator(radius=2)
ao = AllChem.AdditionalOutput(); ao.CollectBitInfoMap()
fp = fpgen.GetSparseCountFingerprint(m, additionalOutput=ao)
info = ao.GetBitInfoMap()   # {bit: ((atomIdx, radius), ...)}

12Maximum Common Substructureبیشینه‌ی زیرساختار مشترک

Given a set of molecules, what's the biggest fragment they all share? That's the Maximum Common Substructure (MCS) — great for aligning series, defining scaffolds, and clustering. FindMCS works on two or more molecules and returns a SMARTS you can reuse as a query.

با داشتن مجموعه‌ای از مولکول‌ها، بزرگ‌ترین قطعه‌ای که همه‌شان مشترک دارند چیست؟ این بیشینه‌ی زیرساختار مشترک (MCS) است — عالی برای هم‌ترازی سری‌ها، تعریف داربست‌ها و خوشه‌بندی. FindMCS روی دو یا چند مولکول کار می‌کند و یک SMARTS برمی‌گرداند که می‌توانید به‌عنوان پرس‌وجو استفاده کنید.

python 
from rdkit import Chem
from rdkit.Chem import rdFMCS

mols = [Chem.MolFromSmiles(s) for s in (
    "O=C(NCc1cc(OC)c(O)cc1)CCCC/C=C/C(C)C",
    "CC(C)CCCCCC(=O)NCC1=CC(=C(C=C1)O)OC",
    "c1(C=O)cc(OC)c(O)cc1")]

res = rdFMCS.FindMCS(mols)
print(res.numAtoms, res.numBonds)   # 10 10
print(res.smartsString)             # reusable SMARTS query
print(res.canceled)                 # False (didn't time out)

# Tune matching: ring-only bonds, complete rings, timeouts ...
res = rdFMCS.FindMCS(mols, ringMatchesRingOnly=True,
                     completeRingsOnly=True, timeout=10)
Useful knobs on FindMCS: atomCompare/bondCompare (match by element vs. anything), ringMatchesRingOnly=True (stop a chain from matching a ring), completeRingsOnly=True (no half rings), and timeout (return the best match found so far). If the search is cut short, res.canceled comes back True. پیچ‌های مفید FindMCS: atomCompare/bondCompare (تطبیق بر اساس عنصر یا هرچیز)، ringMatchesRingOnly=True (جلوگیری از تطبیق زنجیره با حلقه)، completeRingsOnly=True (بدون نیم‌حلقه) و timeout (بازگرداندن بهترین تطبیق یافته‌شده). اگر جستجو نیمه‌کاره قطع شود، res.canceled برابر True بازمی‌گردد.

13Picking Diverse Moleculesانتخاب مولکول‌های متنوع

Sometimes you want a small, maximally diverse subset of a big library — for example to pick compounds for an initial screen. The efficient MaxMin algorithm does exactly this: it greedily picks molecules that are as far apart as possible in fingerprint space.

گاهی زیرمجموعه‌ای کوچک و حداکثر متنوع از یک کتابخانه‌ی بزرگ می‌خواهید — مثلاً برای انتخاب ترکیبات برای غربالگری اولیه. الگوریتم کارآمد MaxMin دقیقاً همین کار را می‌کند: حریصانه مولکول‌هایی را انتخاب می‌کند که در فضای اثرانگشت تا حد امکان از هم دور باشند.

python 
from rdkit import Chem
from rdkit.Chem import rdFingerprintGenerator
from rdkit.SimDivFilters.rdSimDivPickers import MaxMinPicker

fpgen = rdFingerprintGenerator.GetMorganGenerator(radius=3)
with Chem.SDMolSupplier('data/actives_5ht3.sdf') as suppl:
    ms = [m for m in suppl if m is not None]
fps = [fpgen.GetFingerprint(m) for m in ms]

picker = MaxMinPicker()
idx = picker.LazyBitVectorPick(fps, len(fps), 10, seed=23)  # pick 10 diverse
picks = [ms[i] for i in idx]     # the actual diverse molecules

14Drawing & Highlightingرسم و برجسته‌سازی

RDKit produces clean 2D depictions — single images, grids, or highlighted structures. The Draw module covers the quick cases; rdMolDraw2D gives you fine control (SVG or PNG, custom colors, annotations).

RDKit نمایش‌های دوبعدی تمیز تولید می‌کند — تصاویر تکی، شبکه‌ای، یا ساختارهای برجسته‌شده. ماژول Draw موارد سریع را پوشش می‌دهد؛ rdMolDraw2D کنترل دقیق (SVG یا PNG، رنگ‌های سفارشی، حاشیه‌نویسی) به شما می‌دهد.

python 
from rdkit import Chem
from rdkit.Chem import Draw

mol = Chem.MolFromSmiles('CC(=O)OC1=CC=CC=C1C(=O)O')   # aspirin
Draw.MolToFile(mol, 'aspirin.png', size=(400, 300))

# A grid with legends
mols = [Chem.MolFromSmiles(s) for s in ('CCO', 'CCN', 'c1ccccc1')]
img = Draw.MolsToGridImage(mols, molsPerRow=3,
                           legends=['ethanol', 'ethylamine', 'benzene'])
img.save('grid.png')

Highlighting a substructure match

برجسته‌سازی یک تطبیق زیرساختار

A classic use is to draw a molecule with the atoms and bonds of a query lit up. This is where communicating why a molecule matched becomes visual and obvious.

یک کاربرد کلاسیک، رسم مولکول با روشن‌کردن اتم‌ها و پیوندهای یک پرس‌وجو است. اینجاست که انتقال اینکه چرا یک مولکول منطبق شده، دیداری و بدیهی می‌شود.

python 
from rdkit import Chem
from rdkit.Chem.Draw import rdMolDraw2D

mol = Chem.MolFromSmiles('c1cc(F)ccc1Cl')
patt = Chem.MolFromSmarts('ClccccF')
hit_atoms = list(mol.GetSubstructMatch(patt))
hit_bonds = []
for b in patt.GetBonds():
    a1 = hit_atoms[b.GetBeginAtomIdx()]
    a2 = hit_atoms[b.GetEndAtomIdx()]
    hit_bonds.append(mol.GetBondBetweenAtoms(a1, a2).GetIdx())

d = rdMolDraw2D.MolDraw2DCairo(500, 500)   # or MolDraw2DSVG for SVG
rdMolDraw2D.PrepareAndDrawMolecule(d, mol,
    highlightAtoms=hit_atoms, highlightBonds=hit_bonds)
d.FinishDrawing()
open('highlight.png', 'wb').write(d.GetDrawingText())
🖼️ Since 2020.09, PNGs written by MolDraw2DCairo embed the molecule's structure as invisible metadata. You can round-trip it back with Chem.MolFromPNGString(png) — so a saved image doubles as a data file. 🖼️ از نسخه‌ی ۲۰۲۰٫۰۹، PNGهای نوشته‌شده توسط MolDraw2DCairo ساختار مولکول را به‌عنوان فراداده‌ی نامرئی جای می‌دهند. می‌توانید آن را با Chem.MolFromPNGString(png) بازیابی کنید — پس یک تصویرِ ذخیره‌شده هم‌زمان یک فایل داده هم هست.

153D Conformersکانفورمرهای سه‌بعدی

A SMILES describes connectivity, not shape. To get real 3D coordinates — needed for docking, shape comparison, or 3D descriptors — you embed the molecule. The modern default is ETKDGv3, which uses torsion-angle preferences from crystal structures, so you usually don't even need a follow-up force-field cleanup.

یک SMILES اتصال را توصیف می‌کند، نه شکل را. برای به‌دست‌آوردن مختصات سه‌بعدی واقعی — که برای داکینگ، مقایسه‌ی شکل یا توصیفگرهای سه‌بعدی لازم است — مولکول را تعبیه (embed) می‌کنید. پیش‌فرض مدرن ETKDGv3 است که از ترجیحات زاویه‌ی پیچشی در ساختارهای بلوری استفاده می‌کند، پس معمولاً حتی به پاک‌سازیِ بعدی با میدان نیرو هم نیاز ندارید.

⚠️ Always add hydrogens before embedding. Explicit Hs are essential for good geometry. You can strip them again afterward if you don't need them for analysis. ⚠️ همیشه پیش از تعبیه، هیدروژن اضافه کنید. هیدروژن‌های صریح برای هندسه‌ی خوب ضروری‌اند. اگر برای تحلیل به آن‌ها نیاز ندارید، بعداً می‌توانید دوباره حذفشان کنید.
python 
from rdkit import Chem
from rdkit.Chem import AllChem

m = Chem.MolFromSmiles('C1CCC1OC')
mh = Chem.AddHs(m)                    # 1. add Hs (essential!)

params = AllChem.ETKDGv3()
params.randomSeed = 0xf00d           # reproducible
AllChem.EmbedMolecule(mh, params)    # 2. generate a 3D conformer

# Optional: relax with the MMFF94 force field
AllChem.MMFFOptimizeMolecule(mh)

m3 = Chem.RemoveHs(mh)               # 3. drop Hs if you don't need them
print(Chem.MolToMolBlock(m3)[:60])   # now has 3D coordinates

Multiple conformers & RMSD

چند کانفورمر و RMSD

For a flexible molecule you'll want several candidate shapes. Generate many at once with EmbedMultipleConfs, then align them and measure how different they are with RMSD. Set numThreads=0 to use all CPU cores for these embarrassingly parallel jobs.

برای یک مولکول انعطاف‌پذیر، چند شکل نامزد می‌خواهید. با EmbedMultipleConfs چندتا را یک‌جا تولید کنید، سپس آن‌ها را هم‌تراز کرده و با RMSD اندازه بگیرید چقدر متفاوت‌اند. برای استفاده از همه‌ی هسته‌های CPU در این کارهای به‌طرز خجالت‌آوری موازی، numThreads=0 بگذارید.

python 
m = Chem.AddHs(Chem.MolFromSmiles('C1CCC1OC'))
params = AllChem.ETKDGv3()
params.numThreads = 0                 # use all cores

cids = AllChem.EmbedMultipleConfs(m, numConfs=10, params=params)
print(len(cids))                      # 10 conformers

rmslist = []
AllChem.AlignMolConformers(m, RMSlist=rmslist)   # align + get RMSDs
# Optionally relax every conformer with MMFF94
res = AllChem.MMFFOptimizeMoleculeConfs(m, numThreads=0)
Conformer generation is genuinely hard. RDKit's tool is meant for quick, reasonable 3D structures — not a replacement for a dedicated conformational-analysis package. That said, ETKDG is good enough for most everyday purposes. تولید کانفورمر واقعاً دشوار است. ابزار RDKit برای ساختارهای سه‌بعدیِ سریع و معقول در نظر گرفته شده — نه جایگزینی برای یک بسته‌ی اختصاصی تحلیل کانفورماسیونی. با این حال، ETKDG برای بیشتر کاربردهای روزمره به‌اندازه‌ی کافی خوب است.

16Chemical Reactionsواکنش‌های شیمیایی

RDKit can apply reactions to molecules. You describe a transformation with reaction SMARTS (reactants >> products), where atom map numbers like [C:1] track which atom goes where. RunReactants then generates every possible product set.

RDKit می‌تواند واکنش‌ها را روی مولکول‌ها اعمال کند. یک تبدیل را با SMARTSِ واکنش (واکنش‌دهنده‌ها >> محصولات) توصیف می‌کنید، که در آن شماره‌های نگاشت اتمی مانند [C:1] ردگیری می‌کنند کدام اتم به کجا می‌رود. سپس RunReactants هر مجموعه‌ی محصول ممکن را تولید می‌کند.

python 
from rdkit import Chem
from rdkit.Chem import AllChem

# Amide-bond formation: acid + amine -> amide
rxn = AllChem.ReactionFromSmarts(
    '[C:1](=[O:2])-[OD1].[N!H0:3]>>[C:1](=[O:2])[N:3]')

products = rxn.RunReactants((Chem.MolFromSmiles('CC(=O)O'),   # acetic acid
                             Chem.MolFromSmiles('NC')))        # methylamine
print(Chem.MolToSmiles(products[0][0]))   # 'CNC(C)=O'

# One acid with two -OH groups -> two possible products
ps = rxn.RunReactants((Chem.MolFromSmiles('C(COC(=O)O)C(=O)O'),
                       Chem.MolFromSmiles('NC')))
print(len(ps))   # 2
⚠️ Reaction products are not sanitized automatically. Before using them, call prod.UpdatePropertyCache() and/or Chem.SanitizeMol(prod). Also, loose SMARTS can match unintended sites (e.g. an existing amide N); you can shield atoms by setting the _protected property on them so they sit out the reaction. ⚠️ محصولات واکنش به‌طور خودکار پاک‌سازی نمی‌شوند. پیش از استفاده، prod.UpdatePropertyCache() و/یا Chem.SanitizeMol(prod) را فراخوانی کنید. همچنین SMARTSِ شل می‌تواند جایگاه‌های ناخواسته (مثلاً نیتروژن آمیدیِ موجود) را منطبق کند؛ می‌توانید با تنظیم ویژگی _protected روی اتم‌ها آن‌ها را محافظت کنید تا از واکنش کنار بمانند.

Fragmentation: RECAP & BRICS

قطعه‌سازی: RECAP و BRICS

The flip side of building molecules is breaking them apart along realistic, synthetically sensible bonds. RECAP and BRICS both do this — handy for building fragment libraries or generating new ideas by recombining pieces.

روی دیگرِ ساختِ مولکول‌ها، شکستنِ آن‌ها در امتداد پیوندهای واقع‌گرایانه و از نظر سنتزی معقول است. هم RECAP و هم BRICS این کار را می‌کنند — مفید برای ساخت کتابخانه‌های قطعه یا تولید ایده‌های جدید با ترکیب دوباره‌ی قطعات.

python 
from rdkit import Chem
from rdkit.Chem import BRICS

m = Chem.MolFromSmiles('c1ccccc1OCCOC(=O)CC')
frags = BRICS.BRICSDecompose(m)      # unique, synthetically sensible fragments
print(sorted(frags))
# Dummy atoms ([*]) mark where bonds were cut

# BRICS can also recombine fragments into new molecules (a generator)
builder = BRICS.BRICSBuild([Chem.MolFromSmiles(f) for f in frags])
new_mol = next(builder)
new_mol.UpdatePropertyCache(strict=False)   # products need cleanup

17Drug-Likeness & Filtersشبه‌داروبودن و فیلترها

In virtual screening you often want to keep only "drug-like" molecules and toss the trouble-makers. RDKit makes both easy.

در غربالگری مجازی اغلب می‌خواهید فقط مولکول‌های «شبه‌دارو» را نگه دارید و دردسرسازها را دور بریزید. RDKit هر دو را آسان می‌کند.

Lipinski's Rule of 5

قانون پنجِ لیپینسکی

A quick oral-bioavailability heuristic: poor absorption is likely if a molecule breaks more than one of — MW ≤ 500, H-bond donors ≤ 5, H-bond acceptors ≤ 10, LogP ≤ 5. It's a rule of thumb, not a law (plenty of approved drugs break it), but it's a cheap first filter.

یک قاعده‌ی سرانگشتیِ سریع برای دسترسی‌زیستیِ خوراکی: جذب ضعیف محتمل است اگر مولکول بیش از یکی از این‌ها را نقض کند — وزن مولکولی ≤ ۵۰۰، دهنده‌های پیوند هیدروژنی ≤ ۵، گیرنده‌ها ≤ ۱۰، LogP ≤ ۵. این یک قاعده‌ی سرانگشتی است، نه قانون (بسیاری داروهای تأییدشده آن را نقض می‌کنند)، اما یک فیلتر اولیه‌ی ارزان است.

python 
from rdkit import Chem
from rdkit.Chem import Descriptors

mol = Chem.MolFromSmiles('CC(=O)Nc1ccc(O)cc1')   # paracetamol

checks = [
    Descriptors.MolWt(mol)     <= 500,
    Descriptors.NHOHCount(mol) <= 5,    # H-bond donors
    Descriptors.NOCount(mol)   <= 10,   # H-bond acceptors
    Descriptors.MolLogP(mol)   <= 5,
]
passes_ro5 = checks.count(True) >= 3    # allowed at most one violation
print(passes_ro5)                       # True

Screening out unwanted substructures

حذف زیرساختارهای ناخواسته

RDKit's FilterCatalog ships with curated blocklists: PAINS (frequent false-positives in assays), Brenk (toxic/reactive groups), and NIH. You can check one catalog, or all of them at once and read back exactly which rule fired.

‏FilterCatalogی RDKit با فهرست‌های مسدودسازیِ گلچین‌شده همراه است: PAINS (مثبت‌کاذب‌های پرتکرار در سنجش‌ها)، Brenk (گروه‌های سمی/واکنش‌پذیر) و NIH. می‌توانید یک کاتالوگ یا همه را یک‌جا بررسی کرده و دقیقاً بخوانید کدام قاعده فعال شده است.

python 
from rdkit import Chem
from rdkit.Chem.FilterCatalog import FilterCatalog, FilterCatalogParams

mol = Chem.MolFromSmiles('CC(=O)Nc1ccc(O)cc1')

# Check ALL built-in filter sets at once
params = FilterCatalogParams()
params.AddCatalog(FilterCatalogParams.FilterCatalogs.ALL)
catalog = FilterCatalog(params)

if catalog.HasMatch(mol):
    for entry in catalog.GetMatches(mol):
        print(entry.GetProp('FilterSet'), '->', entry.GetDescription())
else:
    print('Clean — no unwanted substructures flagged')

18Tips, Gotchas & Next Stepsنکات، دام‌ها و گام‌های بعدی

  • Chem vs AllChem. Basic stuff (I/O, substructure, cleanup) is in rdkit.Chem; heavier machinery (3D, force fields, reactions) is in rdkit.Chem.AllChem. If the split annoys you, do from rdkit.Chem import AllChem as Chem.
  • Always check for None. Failed parses return None, never an exception — guard every loop.
  • Sanitize after editing. Hand-edited or reaction-generated molecules can look valid but be nonsense; Chem.SanitizeMol catches it.
  • Add Hs before 3D work. Explicit hydrogens are essential for realistic geometry.
  • Pickle for speed. Store frequently-reused molecule sets as pickles instead of re-parsing files.
  • The SSSR caveat. "Smallest set of smallest rings" isn't unique for symmetric molecules — RDKit's GetSymmSSSR returns a symmetric (small, not smallest) set on purpose.
  • Editing in place. For structural surgery use Chem.RWMol; wrap batch edits in a with block or BeginBatchEdit()/CommitBatchEdit().
  • Getting help. Everything is documented in docstrings — help(mol.GetNumAtoms) right in your Python session.
  • ‏Chem در برابر AllChem. کارهای پایه (ورودی/خروجی، زیرساختار، پاک‌سازی) در rdkit.Chem؛ ابزارهای سنگین‌تر (سه‌بعدی، میدان نیرو، واکنش‌ها) در rdkit.Chem.AllChem. اگر این تفکیک آزارتان می‌دهد، from rdkit.Chem import AllChem as Chem بنویسید.
  • همیشه None را بررسی کنید. تجزیه‌های ناموفق None برمی‌گردانند، نه استثنا — از هر حلقه محافظت کنید.
  • پس از ویرایش پاک‌سازی کنید. مولکول‌های دستی‌ویرایش‌شده یا حاصل واکنش می‌توانند معتبر به‌نظر برسند اما مهمل باشند؛ Chem.SanitizeMol آن را می‌گیرد.
  • پیش از کار سه‌بعدی هیدروژن اضافه کنید. هیدروژن‌های صریح برای هندسه‌ی واقع‌گرایانه ضروری‌اند.
  • برای سرعت pickle کنید. مجموعه‌های پرتکرار مولکول را به‌جای تجزیه‌ی دوباره‌ی فایل، به‌صورت pickle ذخیره کنید.
  • هشدار SSSR. «کوچک‌ترین مجموعه‌ی کوچک‌ترین حلقه‌ها» برای مولکول‌های متقارن یکتا نیست — GetSymmSSSRی RDKit عمداً یک مجموعه‌ی متقارن (کوچک، نه کوچک‌ترین) برمی‌گرداند.
  • ویرایش درجا. برای جراحی ساختاری از Chem.RWMol استفاده کنید؛ ویرایش‌های دسته‌ای را در بلوک with یا BeginBatchEdit()/CommitBatchEdit() بپیچید.
  • گرفتن کمک. همه‌چیز در docstringها مستند شده — help(mol.GetNumAtoms) درست در نشست پایتون خودتان.
